# 智慧进化 — 两级提炼架构

## 一句话

- **每轮提炼（L1）**：及时捕捉有没有新的重要记忆  
- **会话复盘（L2）**：碎片怎么组织、合并、升级  
- **Wiki 编译**：哪些内容进入企业知识库（ingest 管线，独立）

```
Chat 原始记录 (PG)
    ↓ L1 每轮异步
轻量记忆提取 (project / preference / decision)
    ↓ L2 会话关闭 / 定时 / 手动
会话复盘 (总结 · 合并 · 去重 · 冲突 · Wiki 建议)
    ↓
PostgreSQL memories + memory_events
    ↓
Qdrant 向量 → Chat 智慧召回 + 智慧进化页
```

## 第一级：每轮轻量提炼（已实现）

**触发**：每次 `POST /chat` 回答返回后，`BackgroundTasks` 异步执行。

**提示词**：`evolution.l1_memory_extractor`（原 `memory_extractor.py` 内嵌，已外置）

**输入**：
- 已有活跃智慧（全文列表，上限 50 条）
- 最近 3 轮对话（6 条消息，每条 ≤500 字）
- 当前一轮：用户问题 + 助手回答（≤2000 字）

**规则**：见 `prompts.yaml` → `evolution.l1_memory_extractor`（日期→decision、偏好→preference、记住了→必提取等）。

**作用**：解决「有没有新的重要记忆」。

## 第二级：会话级复盘（已实现）

**触发**（任选）：
1. `POST /api/v1/orgs/{org}/memories/recap-session` — 指定 session_id
2. 删除会话时 `DELETE .../chat/sessions/{id}?recap=true`（**WebUI 弹窗**：勾选「L2 会话复盘」后删除，展示摘要 / 更新数 / Wiki 建议）
3. `POST /api/v1/orgs/{org}/memories/recap-batch` — 批量
4. `python scripts/recap_sessions.py` — 定时任务

**输入**：
- 整段会话记录（最多 40 轮，每条 ≤800 字）
- 全部活跃智慧 + 本会话已沉淀智慧

**输出**：
- `summary` — 会话摘要
- `memories` — create / update（合并升级）
- `archives` — 去重归档（status=archived）
- `conflicts` — 冲突与建议_resolution
- `wiki_suggestions` — 建议进 Wiki → **写入知识候选池**（见 [5-知识候选池.md](./5-知识候选池.md)）

**作用**：解决「这些记忆之间怎么组织、合并、升级」。

L1/L2 在写入 `memories` 的同时，对符合门槛的条目**双写** `knowledge_candidates`，供 Wiki 晋升审核。

自动批准 vs 人工审核的分界与三档运营配置，见 **[6-自动与人工审核分界线.md](./6-自动与人工审核分界线.md)**。

## Wiki 编译（独立）

上传资料 → 编译 → Wiki / 实体图谱。与 Chat 提炼并行，负责企业知识资产化。

Chat 侧晋升路径：**候选池 → Resolver → Wiki 编译**（P1 已实现自动写 Wiki，见 [5-知识候选池.md](./5-知识候选池.md)）。

## 运维

```bash
# 向量同步
python scripts/sync_memory_vectors.py

# 会话复盘（cron 示例：每天 2:00；跳过 recapped_at 之后无新消息的会话）
python scripts/recap_sessions.py --org demo --limit 20 --idle-hours 24

# 单会话复盘
curl -X POST http://localhost:8006/api/v1/orgs/demo/memories/recap-session \
  -H 'Content-Type: application/json' \
  -d '{"session_id":"<uuid>"}'
```
