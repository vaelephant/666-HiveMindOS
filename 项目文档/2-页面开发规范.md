# 页面开发规范

> 适用于 `webui/`（Next.js 16 App Router + React 19 + Tailwind v4）。
> 目标：新页面长得一样、改起来一样、不踩同样的坑。照着抄即可。

---

## 1. 技术栈基线

| 项目 | 约定 |
| --- | --- |
| 框架 | Next.js 16 App Router |
| UI | React 19，页面默认写 `'use client'` |
| 样式 | Tailwind v4，**只用 `globals.css` 里定义的设计 token**，不写死颜色 |
| 图标 | `lucide-react`，统一 `strokeWidth={1.5}`～`1.75` |
| 语言 | 界面文案用中文 |
| 校验 | 提交前必须过 `npm run lint`（即 `tsc --noEmit`） |

---

## 2. 目录结构（放对位置）

```
webui/src/
├── app/
│   ├── (platform)/<模块>/<页面>/page.tsx   # 路由页面，按路由组划分
│   ├── (fleet)/<页面>/page.tsx
│   └── api/<模块>/.../route.ts             # 后端代理路由
├── components/<模块>/<组件>.tsx            # 页面拆出来的展示组件
└── lib/
    ├── <模块>-api.ts                       # 前端 API 调用封装
    └── <模块>-types.ts                     # 该模块的 TypeScript 类型
```

规则：

- **路由组**（`(platform)`、`(fleet)`）只决定布局，不出现在 URL 里。新页面先确认归属哪个组。
- 页面文件只放在 `page.tsx`；复杂 UI 拆到 `components/<模块>/` 下，**不要把 200+ 行 JSX 全塞进 page**。
- 数据类型集中放 `lib/<模块>-types.ts`，**禁止在组件里散落定义重复类型**。

---

## 3. 页面文件写法

每个 `page.tsx` 遵循固定骨架：

```tsx
'use client';

import { useEffect, useState } from 'react';
import { Loader2 } from 'lucide-react';
import { getSomething } from '@/lib/xxx-api';
import type { Something } from '@/lib/xxx-types';

export default function XxxPage() {
  const [data, setData] = useState<Something | null>(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState<string | null>(null);

  useEffect(() => {
    setLoading(true);
    setError(null);
    getSomething()
      .then(setData)
      .catch((e: Error) => setError(e.message))
      .finally(() => setLoading(false));
  }, []);

  // 三态渲染：loading / error / empty / 正常
  ...
}
```

要点：

1. **导入用 `@/` 别名**，不写相对路径 `../../../`。
2. **状态命名统一**：`loading` / `error`（`string | null`）/ 数据本身。`error` 存消息文本，不是 boolean。
3. `useEffect` 拉数据：进入时 `setLoading(true); setError(null)`，结束 `finally(setLoading(false))`。
4. 用到 `useSearchParams` / `useRouter` 读 URL 的页面，**必须用 `<Suspense>` 包一层内容组件**（见 wiki 页面写法），否则 build 会报错。
5. URL 是状态的一部分：选中项、分类、tab 等用 `router.replace(..., { scroll: false })` 同步到 query，刷新可还原。

## 4. 数据请求：统一走 `lib/<模块>-api.ts`

**组件里不直接写 `fetch`。** 所有请求封装成函数放在 api 文件，复用同一个请求帮手：

```ts
async function req<T>(url: string, init?: RequestInit): Promise<T> {
  const res = await fetch(url, init);
  if (!res.ok) {
    const text = await res.text().catch(() => res.statusText);
    throw new Error(text || `HTTP ${res.status}`);
  }
  return res.json() as Promise<T>;
}

export async function listXxx(): Promise<Xxx[]> {
  const data = await req<{ items: Xxx[] }>(`/api/xxx`);
  return data.items;
}
```

约定：

- 函数命名：`listXxx` / `getXxx` / `createXxx` / `deleteXxx` / `updateXxx`。
- 返回值带类型，从 `lib/<模块>-types.ts` 导入。
- 错误用 `throw new Error(message)`，页面层 `catch` 后放进 `error` 状态展示。
- 真正的外部/后端请求放在 `app/api/.../route.ts` 里代理，前端只调本地 `/api/...`。

## 5. 三态 + 空态（必做）

每个会异步加载数据的区域都要覆盖四种情况，缺一不可：

| 状态 | 表现 |
| --- | --- |
| 加载中 | `<Loader2 className="size-4 animate-spin" />` + “加载中…” |
| 出错 | `text-status-error` 文案，展示 `error` 内容 |
| 空数据 | 居中提示框，引导用户下一步操作 |
| 正常 | 渲染内容 |

空态参考样式（虚线卡片 + 图标 + 引导文案）：

```tsx
<div className="flex flex-col items-center justify-center rounded-2xl border border-dashed border-shell-border bg-shell-panel/50 p-10 text-center">
  <p className="mt-4 text-[15px] font-medium text-shell-text">还没有数据</p>
  <p className="mt-1 text-[13px] text-shell-muted">引导用户的下一步操作</p>
</div>
```

## 6. 样式与设计 token（严格）

**禁止写死颜色**（不要 `text-gray-500`、`bg-[#xxx]`、`text-purple-600`）。统一用 `globals.css` 的 token：

| 用途 | token |
| --- | --- |
| 页面底色 / 面板 | `bg-shell-bg` / `bg-shell-panel` |
| 边框 | `border-shell-border`，hover 用 `border-shell-border-hover` |
| 文字主/次/弱 | `text-shell-text` / `text-shell-subtext` / `text-shell-muted` |
| 品牌主色 | `bg-brand-primary` / `text-brand-primary`，主色上的文字 `text-brand-on-primary` |
| 状态色 | `text-status-success` / `warning` / `error` / `info` |

其它约定：

- 圆角：卡片 `rounded-2xl`、按钮/输入 `rounded-xl`、标签 `rounded-lg/full`。
- 卡片基底：`rounded-2xl border border-shell-border bg-shell-panel shadow-sm`（或直接用工具类 `surface-card`）。
- 可滚动区域加 `custom-scrollbar` 类。
- 字号偏小、精确：正文 `text-[13px]`/`text-[14px]`，说明/标签 `text-[11px]`/`text-[12px]`。
- 间距用 `gap-*` / `space-y-*`，保持与现有页面一致的节奏（`gap-4`、`p-3`、`px-4 py-2.5`）。
- 深色模式自动生效（token 已在 `.dark` 定义），**不要单独写 `dark:` 颜色覆盖**。

可直接复用的工具类（见 `globals.css`）：`surface-card`、`btn-primary`、`input-field`、`nav-item-active` / `nav-item-idle`。

---

## 7. 组件拆分

- 列表项、卡片、详情区等可复用片段拆成独立组件，放 `components/<模块>/`。
- 组件 props 用具名 `type`，避免 inline 大对象类型。
- 交互按钮用 `<button type="button">`，明确 `type` 避免误触发表单提交。
- 列表渲染务必给稳定 `key`（用 `id` / `path`，不要用 index）。

## 8. 提交前自检清单

- [ ] 放对路由组与目录，page 不臃肿（复杂 UI 已拆组件）。
- [ ] 没有写死颜色，全部用设计 token；深浅色都正常。
- [ ] loading / error / empty / 正常 四态都覆盖。
- [ ] 数据请求走 `lib/<模块>-api.ts`，类型来自 `lib/<模块>-types.ts`。
- [ ] 用了 `useSearchParams` 的页面已用 `<Suspense>` 包裹。
- [ ] `npm run lint` 通过，无 TS 报错。
- [ ] 文案为中文，图标 stroke 统一。
