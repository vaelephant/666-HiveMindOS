# YAML 工作流编排

> 轻量多步编排：串联 `automation.*` 与 `tool.*`，支持 `when` 条件分支。

---

## 一句话

**工作流 = 把多个自动化步骤串成一条管线**，比单条 cron 任务更适合「先复盘、再解析、再编译」这类有顺序、有条件的流程。

与 **定时运维**（`/tasks/ops`）的区别：

| | 定时运维 | 工作流 |
|---|---------|--------|
| 粒度 | 单条 automation job | 多步 YAML 序列 |
| 条件 | 无 | `when: "$step.field >= N"` |
| 存储 | `automation_jobs` | `workflows`（SQLite） |

---

## YAML 格式

```yaml
id: nightly_knowledge_pipeline   # 小写+下划线，org 内唯一
label: 夜间知识管线
description: 会话复盘 → 解析 → 编译 → 巡检
category: wiki                   # memory | wiki | mixed
cron_hint: "0 2 * * *"           # 供外部 cron 参考，引擎不自动调度
enabled: true
steps:
  - id: recap
    action: automation.recap_sessions
    params:
      idle_hours: 24
      limit: 10
  - id: resolve
    action: automation.resolve_candidates
    params:
      limit: 30
  - id: compile
    action: automation.compile_candidates
    when: "$resolve.approved >= 1"
    params:
      limit: 20
  - id: lint
    action: automation.lint_wiki
```

### action 命名空间

| 前缀 | 说明 | 示例 |
|------|------|------|
| `automation.*` | 调用 `automations.yaml` 中已注册的 job | `automation.daily_digest` |
| `tool.*` | 调用自主任务 Tool（无人工门拦截） | `tool.search_wiki` |

### when 条件

复用任务引擎语法，检查**前序步骤**结果字段：

```yaml
when: "$resolve.approved >= 1"
```

### params 引用

参数值可用 `$step_id.field` 引用前序步骤输出：

```yaml
params:
  limit: "$recap.sessions_recapped"
```

## 定时调度

### 方式一：进程内调度（开发 / 单实例部署）

根目录 `.env`：

```bash
WORKFLOW_SCHEDULER_ENABLED=true
WORKFLOW_SCHEDULER_TICK_SECONDS=60   # 默认 60
```

后端启动后每分钟扫描 `schedule_enabled=1` 且含 `cron_hint` 的工作流。

### 方式二：系统 cron（生产推荐）

```bash
# 每分钟检查到期工作流
* * * * * cd /path/to/backend && python scripts/run_scheduled_workflows.py --org demo
```

### YAML / UI 开关

```yaml
cron_hint: "0 2 * * *"
schedule_enabled: true      # 也可在 WebUI 点时钟图标切换
schedule_user_id: demo    # 可选，执行时使用的 user_id
```

- 时区：**Asia/Shanghai**（与用量统计一致）
- 防重复：同一 cron 槽位只触发一次（`last_cron_at`）
- 运行记录 `trigger=cron`，审计 action 仍为 `workflow.run`

## API

| 方法 | 路径 | 说明 |
|------|------|------|
| PATCH | `/orgs/{org}/workflows/{id}/schedule` | 开启/关闭定时 `{ enabled, user_id }` |
|------|------|------|
| GET | `/orgs/{org}/workflows` | 列表 |
| GET | `/orgs/{org}/workflows/templates` | 模板 |
| GET | `/orgs/{org}/workflows/{id}` | 详情 + YAML |
| POST | `/orgs/{org}/workflows` | 从 YAML 创建 |
| PUT | `/orgs/{org}/workflows/{id}` | 更新 YAML |
| POST | `/orgs/{org}/workflows/{id}/run` | 手动执行 |
| GET | `/orgs/{org}/workflows/runs` | 运行历史 |

WebUI：`/workflows`

存储：`storage/workflows.db`（`config.WORKFLOW_DB`）
